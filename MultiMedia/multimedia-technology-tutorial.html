Module 01 · Types
Types of CMS Platforms
CMS platforms are categorised by how content is stored, managed, and delivered to the end user.
🔗
Traditional / Coupled CMS
Frontend and backend are tightly linked. Content is created and displayed by the same system. Fast to set up for simple websites.

Examples: WordPress (43% of all websites), Joomla, Drupal

Best for: Blogs, simple websites
🔓
Headless / Decoupled CMS
Content is stored in a backend and delivered via API to any frontend — web, mobile, IoT, smart TV. Maximum flexibility.

Examples: Contentful, Strapi, Sanity

Best for: Omnichannel, complex apps
☁️
Cloud / SaaS CMS
Fully hosted and managed by the vendor. No server maintenance required — you simply subscribe and publish.

Examples: WordPress.com, Squarespace, Wix

Best for: Non-technical users
🏢
Enterprise CMS
High-end commercial platforms built for large organizations needing advanced workflows, compliance, and integrations.

Examples: Adobe Experience Manager, Sitecore, Kentico

Best for: Large enterprises

  • ✏️

Module 01 · Deployment
CMS Deployment Strategy
Deploying a CMS follows a structured lifecycle. Skipping phases leads to costly rework and security vulnerabilities.
PHASE 1
Define Requirements
Identify content types, team size, traffic projections, compliance needs, integrations required, and budget.
PHASE 2
Evaluate & Select Platform
Compare CMS options against requirements. Consider open-source vs. commercial, SaaS vs. self-hosted, headless vs. traditional.
PHASE 3
Design Architecture & Infrastructure
Set up hosting environment, configure SSL/firewall/backups, establish Git workflow and CI/CD pipeline, plan CDN strategy.
PHASE 4
Content Migration
Content Audit → Data Mapping → Migration Scripts → QA. Always set up 301 redirects for changed URLs to preserve SEO.
PHASE 5
Testing & Launch
Functionality testing, performance benchmarking, security audit, user acceptance testing (UAT), then go live.

Module 02 · HTML & CSS
HTML & CSS — The Foundation
Every web page is built on HTML (structure) and CSS (presentation). Understanding their roles is the first step in web development.
📄
HTML — HyperText Markup Language
Defines the structure and content of a web page using elements (tags). Introduced by Tim Berners-Lee in 1991. HTML5 added native support for audio, video, canvas, local storage, and semantic elements.
HTML5
<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <h1>Heading</h1> <p>Paragraph text</p> <video src="clip.mp4" controls></video> </body> </html>
🎨
CSS — Cascading Style Sheets
Controls the visual presentation: colours, fonts, layouts, spacing, and animations. CSS3 introduced flexbox, grid, transitions, and media queries for responsive design.
CSS3
body { font-family: 'Segoe UI', sans-serif; background: #f5f5f5; } h1 { color: #1a5276; font-size: 2rem; animation: fadeIn 0.5s ease; } /* Responsive layout */ @media (max-width: 768px) { .grid { grid-template-columns: 1fr; } }

JavaScript is the only programming language that runs natively in web browsers, enabling dynamic behaviour, user interaction, and real-time data updates without page reloads.
🖱️
DOM Manipulation

Compression &
Decompression
Algorithms

How multimedia data is reduced in size for efficient storage and transmission, and reconstructed for playback.
Module 03 · Lossless Methods
Lossless Compression
Lossless compression reduces file size while guaranteeing perfect, bit-for-bit reconstruction of the original data. Not a single byte of information is lost.
ORIGINAL: 12 units
→ compress →
ENCODED: 4 tokens (RLE)
→ decompress →
RESTORED: 12 units (identical)
Run-Length Encoding (RLE)
Replaces consecutive repeated values with a single value + count. AAAA → 4A. Used in BMP, TIFF, fax machines.
Huffman Coding
Assigns shorter binary codes to more frequent symbols. Common characters get fewer bits. Foundation of ZIP, PNG, and DEFLATE compression.
LZW (Lempel-Ziv-Welch)
Builds a dictionary of repeated patterns during encoding. Used in GIF, TIFF, and PDF compression. Decompressor rebuilds the same dictionary to reverse it.
💡
When to use lossless compression
Use lossless for logos, icons, screenshots, medical images, legal documents, and any content where perfect accuracy is non-negotiable. Formats: PNG, FLAC, ZIP, TIFF
Module 03 · Lossy Methods
Lossy Compression
Lossy compression permanently discards data that the human perceptual system is unlikely to notice, achieving much higher compression ratios than lossless methods — at the cost of some quality.
🖼️
DCT — Discrete Cosine Transform
The algorithm behind JPEG. Converts image blocks into frequency components, then discards high-frequency (fine detail) data that eyes are less sensitive to. At low quality settings, "blocking" artefacts appear where blocks meet.

Also used in: MPEG video, MP3 audio (DCT variant).
🎵
Perceptual / Psychoacoustic Coding
Used in MP3, AAC, OGG. Exploits hearing limitations — we cannot hear quiet sounds masked by louder simultaneous sounds. The codec removes inaudible audio data, reducing file size by 90%+ vs. raw WAV with minimal perceived quality loss.
🎬
Inter-frame Compression
Used in MPEG, H.264, H.265. Only stores the differences between consecutive video frames — not full frames. A reference "keyframe" is stored periodically; intermediate frames only record what changed. Reduces video file sizes by 100:1 or more.
🌊
Wavelet Compression
Used in JPEG 2000, WebP. Analyses the image at multiple scales — preserving important large-scale structure while discarding fine-scale noise. Avoids blocking artefacts, and supports progressive image loading.

Module 04 · Image Formats
Image File Formats
Choosing the right image format affects file size, visual quality, browser compatibility, and whether transparency is supported.
.JPEG
Lossy Raster
Photographs, complex gradients. Best for web delivery. Doesn't support transparency.
.PNG
Lossless Raster
Logos, screenshots, UI elements. Supports full transparency (alpha channel).
.WebP
Lossy + Lossless
Google format. 25–35% smaller than JPEG/PNG. Supports transparency and animation.
.SVG
Vector (XML)
Infinitely scalable without pixelation. Ideal for icons, logos, illustrations, and responsive graphics.
.GIF
Lossless Raster
Simple animations and graphics. Limited to 256 colours. Widely replaced by WebP and video for animations.
.TIFF
Lossless Raster
Professional archival and printing. Supports multiple pages and layers. Very large file sizes.
.AVIF
Lossy + Lossless
Next-generation format. Better quality per byte than WebP. Growing browser support.
.RAW
Unprocessed
Camera sensor data before processing. Maximum editing flexibility. Requires special software to open.

🎚️
Professional Workflow Tip
Always record and edit in WAV or AIFF (lossless), then export to MP3 or AAC for distribution. Never re-compress an already-lossy file — quality degrades with each generation.
Module 04 · Video Formats
Video File Formats & Codecs
A video file has two components: a container (the file format wrapping everything) and a codec (the algorithm that compresses/decompresses the video stream).
📦 Containers (File Extensions)
.MP4Universal — plays everywhere. Supports H.264, H.265 codecs.
.MOVApple/Final Cut Pro native. High quality, large files.
.MKVOpen standard. Supports multiple audio tracks, subtitles.
.WebMGoogle's open web format — efficient for HTML5 video.
.AVIOlder Windows format. Large files, broad compatibility.
⚙️ Video Codecs (Compression Engines)
H.264Most widely used. Excellent quality-to-size ratio. Supported on every device.
H.265 (HEVC)50% better compression than H.264 at the same quality. Used for 4K/HDR.
VP9Google's open-source codec. Used by YouTube. Similar to H.265.
AV1Next-gen open-source. 30% better than VP9. Slow to encode but best quality.
ProResApple's near-lossless production codec. Used in professional editing.
