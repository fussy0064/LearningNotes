Topic 01

History of Java

Java was created by James Gosling in the early 1990s at Sun Microsystems. The project team — Gosling, Mike Sheridan, and Patrick Naughton — initially wanted to build a language for digital devices like set-top boxes and televisions.

They considered C++ but abandoned it due to high memory requirements and platform dependency. Instead, in 1991, they launched the "Green" project to build a new language called Greentalk (.gt extension), later renamed OAK — named after the oak tree outside Gosling's office.

🌳

OAK (1991)

Named after the oak tree. Already trademarked by Oak Technologies, so the name had to change.

☕

JAVA (1995)

Renamed to Java — a type of delicious coffee from Java Island, Indonesia that Gosling was drinking at the time.

🏢

Oracle (2010)

Oracle acquired Sun Microsystems in January 2010, gaining control of Java, Solaris, and MySQL.

🌍

Today

Java runs on 3+ billion devices across Windows, Mac OS, UNIX and more. Write Once, Run Anywhere (WORA).

Why Use Java?
🛡️

Robust

🔄

OOP

Object-oriented design promotes code reuse and clear program structure.

🆓

Free & Open

🗑️

Auto GC

💻

Platform Independent

Runs on Linux, Windows, Mac OS via the JVM.

Architecture of Java

Java architecture defines how Java programs are created, compiled, and run. It has three main layers: JDK (outermost), JRE (middle), and JVM (innermost).

Translates bytecode → machine code. Enables WORA.

How Java Code Runs
  • 1 Write source code in a .java file
  • 2 Javac (Java Compiler) compiles it into .class bytecode
  • 3 JVM Class Loader loads the .class file into memory
  • 4 Interpreter converts bytecode line-by-line to machine code
  • 5 JIT Compiler optimises frequently-used code for faster execution
  • 6 Program executes on the hardware. Garbage Collector frees unused memory.
💡 WORA — Write Once, Run Anywhere
Because bytecode runs on the JVM (not directly on hardware), the same .class file runs on Windows, Mac, Linux, or any platform that has a JVM installed.

Topic 04

If Statements

Java provides three forms of the if statement for decision-making. They evaluate Boolean expressions to direct program flow.

1️⃣

Simple if

Executes a block only when the condition is true. No alternative block.

2️⃣

if…else

Provides an alternative (else) block that runs when the condition is false.

3️⃣

if…else if…else

Tests multiple conditions in sequence. The matching block executes.

Program: Compare Two Numbers (Simple if)
Program3.java — Simple if
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        if (a > b) {
            System.out.println("a Is Greater");
        }
        if (b > a) {
            System.out.println("b Is Greater");
        }
        if (a == b) {
            System.out.println("a and b are equal");
        }
    }
}
▶ OUTPUT
b Is Greater
Program: Largest of Two Input Numbers (if-else)

        }
    }
}

Topic 05

Switch Case Statement

The switch statement tests an expression against multiple case values. When a match is found the corresponding block executes. A break statement ends the switch. Without it, execution falls through to the next case. A default block runs when no case matches.

Switch Syntax
switch (expression) {
    case value1:
        // code to execute
        break;    // optional — stops fall-through
    case value2:
        // code to execute
        break;
    default:
        // code if no case matches
}
Program: Module Code Lookup
SwitchDemo.java — Module Name from Code
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        System.out.println("Enter Module Code");
        String code = input.nextLine();

        switch (code) {
            case "ITU07313":
                System.out.println("Data Structure and Algorithm");
                break;
            case "ITU07312":
                System.out.println("Programming in Java");
                break;
            case "ITU07212":
                System.out.println("Programming in C++");
                break;
            default:
                System.out.println("Unknown Module");
        }
    }
}
▶ OUTPUT (sample — entering ITU07312)
Enter Module Code ITU07312 Programming in Java
⚠️ Fall-Through Warning
If you omit break, Java continues executing subsequent cases even if they don't match. This can cause unexpected results. Always add break unless fall-through is intentional.

Topic 06

For Loop & While Loop

🔢

for loop

Best when the number of iterations is known. init, condition, and increment are all inside the header.

🔄

while loop

Best when iterations depend on a condition. Init is outside, condition is checked before each iteration.

⤵️

do-while loop

Executes the body at least once. Condition is checked after the first execution.

for Loop — Syntax & Examples
ForLoop.java — Print 1 to 5
// Syntax: for (init; condition; incr/decr)

for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}
▶ OUTPUT
1 2 3 4 5
Stars.java — Triangle Pattern (Nested for loop)
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("*");
    }
    System.out.print("\n");
}
▶ OUTPUT
* ** *** **** *****
InvertedTriangle.java — Inverted Star Pattern
for (int i = 0; i <= 8; i++) {
    for (int j = 1; j <= 8 - i; j++) {
        System.out.print("*");
    }
    System.out.print("\n");
}
▶ OUTPUT
******** ******* ****** ***** **** *** ** *
while Loop — Syntax & Example
WhileLoop.java — Print 1 to 5
int n = 1;
while (n <= 5) {
    System.out.println(n);
    n++;    // increment must be inside the loop body
}
▶ OUTPUT
1 2 3 4 5
do-while Loop — Syntax & Example
DoWhile.java — Always executes at least once
int i = 1;
do {
    System.out.println(i);
    i++;
} while (i <= 5);  // note: semicolon after while
Key Difference
for / while: condition checked before the first iteration — body may never run.
do-while: condition checked after the first iteration — body always runs at least once.

Topic 07

Break & Continue Statements

🛑

break

Immediately exits the loop (or switch). No more iterations occur after a break.

⏩

continue

Skips the remainder of the current iteration and jumps to the next iteration.

Program: break — Stop Loop at Iteration 3
BreakDemo.java
for (int i = 1; i <= 8; i++) {
    System.out.println("Iteration No " + i);
    if (i == 3) {
        break;   // exits loop when i equals 3
    }
}
▶ OUTPUT
Iteration No 1 Iteration No 2 Iteration No 3
Program: continue — Skip Iterations 4 and 5
ContinueDemo.java — Print 1,2,3,6,7,8 (skip 4 and 5)
for (int i = 1; i <= 8; i++) {
    if (i == 4 || i == 5) {
        continue;  // skip printing 4 and 5
    }
    System.out.println(i);
}
▶ OUTPUT
1 2 3 6 7 8

Instance, Static & Local Variables

TypeWhere DeclaredDefault ValueAccess
LocalInside method/blockNone — must initOnly within that method/block
InstanceInside class, outside method; no staticYes (0, false, null)Via object
StaticInside class with static keywordYes (0, false, null)Direct (no object needed)
Program: Demonstrating Variable Types
VariableTypes.java
package AllPrograms;
public class VariableTypes {

    // ── INSTANCE variable (belongs to object)
    int instanceVar = 100;

    // ── STATIC variable (belongs to class)
    static String schoolName = "CBE";

    void showLocal() {
        // ── LOCAL variable (only lives inside this method)
        int localVar = 50;
        System.out.println("Local: "    + localVar);
        System.out.println("Instance: " + instanceVar);
        System.out.println("Static: "   + schoolName);
    }

    public static void main(String[] args) {
        VariableTypes obj = new VariableTypes();
        obj.showLocal();

        // Static variable accessible without object:
        System.out.println("Static direct: " + schoolName);
    }
}
▶ OUTPUT
Local: 50 Instance: 100 Static: CBE Static direct: CBE
✅ Key Rules Summary
• Local: No default value — you MUST assign before use. No access modifiers allowed.
• Instance: Gets memory when object is created. Accessible via object reference.
• Static: Shared across all objects. Accessible directly without creating an object.
• Constants: Declared with final. Value cannot change after assignment.

Topic 09

Classes & Objects

A class is a blueprint for creating objects — it groups variables (state) and methods (behaviour) together. An object is an instance of a class — a real entity created from the blueprint.

🎨

State

Object properties — represented by variables (e.g., car colour, speed).

⚡

Behaviour

Actions an object can perform — represented by methods (e.g., accelerate, brake).

🏷️

Identity

A unique name that distinguishes one object from another.

Static Method Example
StaticMethod.java — Average with static function + parameters
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Static method — called without creating an object
    public static void average(float m, float g, float e) {
        float avg = (m + g + e) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        float math, geog, engl;
        Scanner input = new Scanner(System.in);

        System.out.println("Enter Mathematics Marks");
        math = input.nextFloat();
        System.out.println("Enter Geography Marks");
        geog = input.nextFloat();
        System.out.println("Enter English Marks");
        engl = input.nextFloat();

        average(math, geog, engl);  // called without object
    }
}
▶ OUTPUT (sample: 10, 20, 40)
The Average is 23.333334
Instance Method Example
InstanceMethod.java — Average via object
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Instance method — requires an object to call
    public void average(float m, float g, float e) {
        float avg = (m + g + e) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        float math, geog, engl;

        System.out.println("Enter Mathematics Marks");
        math = input.nextFloat();
        System.out.println("Enter Geography Marks");
        geog = input.nextFloat();
        System.out.println("Enter English Marks");
        engl = input.nextFloat();

        // Create an object first, then call the method
        AllProgramsClass callavg = new AllProgramsClass();
        callavg.average(math, geog, engl);
    }
}
Constructor Example

A constructor is a special method with the same name as the class and no return type. It is automatically called when an object is created with new.

ConstructorDemo.java — Constructor with parameters
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Constructor — same name as class, no return type
    public AllProgramsClass(float a, float b, float c) {
        float avg = (a + b + c) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        float math, geog, engl;
        
        System.out.println("Enter Mathematics Marks"); math = input.nextFloat();
        System.out.println("Enter Geography Marks");  geog = input.nextFloat();
        System.out.println("Enter English Marks");    engl = input.nextFloat();

        // Creating the object calls the constructor automatically
        AllProgramsClass obj = new AllProgramsClass(math, geog, engl);
    }
}
▶ OUTPUT (entering 10, 10, 10)
The Average is 10.0
📌 Constructor Rules
1. Constructor name must be identical to the class name.
2. No explicit return type (not even void).
3. Cannot be abstract, static, final, or synchronized.

Topic 10

Arrays

A Java array is an object that stores elements of the same data type in contiguous memory. It has a fixed size set at creation (instantiation). Arrays start at index 0.

📏

1D Array

Linear storage — elements arranged in a single row. Accessed with one index: arr[i].

🔲

2D Array

Matrix storage — rows and columns. Accessed with two indices: arr[row][col].

1D Array — Declaration & Initialization
Array1D.java — Declaration, Initialization, and Access
// Method 1: Separate declaration and instantiation
int data[];
data = new int[7];
data[0] = 100;
data[1] = 234;
// ... etc

// Method 4: Declare + instantiate + initialise in one line
int data[] = {100, 234, 100, 4, 19, 100, 345};
Program: 1D Array Operations
Array1DProgram.java — Store, Access, Update, Display All
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {

        int data[] = new int[]{100, 234, 100, 4, 19, 100, 345};

        // Access 3rd element (index 2)
        System.out.println("3rd Element: " + data[2]);

        // Update element at index 1 (value 234 → 5000)
        data[1] = 5000;

        // Display all elements with for-loop
        System.out.println("All Elements:");
        for (int i = 0; i <= 6; i++) {
            System.out.println(data[i]);
        }
    }
}
▶ OUTPUT
3rd Element: 100 All Elements: 100 5000 100 4 19 100 345
2D Array — Declaration & Program
Array2DProgram.java — 3×5 Matrix Operations
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {

        int data[][] = new int[][]{{100,234,100,4,19},
                                     {345,45,67,90,1000},
                                     {45,67,89,69,35}};

        // Display 3rd element (index [0][2])
        System.out.println("3rd Element: " + data[0][2]);

        // Display element at [2][0]
        System.out.println("data[2][0]: "  + data[2][0]);

        // Update 1000 → 5000 at [1][4]
        data[1][4] = 5000;

        // Display all with nested for-loop
        System.out.println("All Elements:");
        for (int i = 0; i <= 2; i++) {
            System.out.println("Row " + (i+1));
            for (int j = 0; j <= 4; j++) {
                System.out.print(" " + data[i][j]);
            }
            System.out.println();
        }
    }
}
▶ OUTPUT
3rd Element: 100 data[2][0]: 45 All Elements: Row 1 100 234 100 4 19 Row 2 345 45 67 90 5000 Row 3 45 67 89 69 35
✅ Array Quick Reference
• Arrays start at index 0, not 1
• Size is fixed after creation — use ArrayList for dynamic sizing
• 2D array data[3][5] = 3 rows, 5 columns = 15 total elements
• Access with data[row][col] for 2D arrays
