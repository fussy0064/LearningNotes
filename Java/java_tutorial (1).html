Topic 01

History of Java

Java was created by James Gosling in the early 1990s at Sun Microsystems. The project team — Gosling, Mike Sheridan, and Patrick Naughton — initially wanted to build a language for digital devices like set-top boxes and televisions.

They considered C++ but abandoned it due to high memory requirements and platform dependency. Instead, in 1991, they launched the "Green" project to build a new language called Greentalk (.gt extension), later renamed OAK — named after the oak tree outside Gosling's office.

🌳

OAK (1991)

Named after the oak tree. Already trademarked by Oak Technologies, so the name had to change.

☕

JAVA (1995)

Renamed to Java — a type of delicious coffee from Java Island, Indonesia that Gosling was drinking at the time.

🏢

Oracle (2010)

Oracle acquired Sun Microsystems in January 2010, gaining control of Java, Solaris, and MySQL.

🌍

Today

Java runs on 3+ billion devices across Windows, Mac OS, UNIX and more. Write Once, Run Anywhere (WORA).

Why Use Java?
🛡️

Robust

Built-in error handling with try-catch blocks. Safer and more reliable execution.

🔄

OOP

Object-oriented design promotes code reuse and clear program structure.

🆓

Free & Open

Completely open-source with no licensing cost.

🗑️

Auto GC

Automatic Garbage Collection frees memory no longer in use.

💻

Platform Independent

Runs on Linux, Windows, Mac OS via the JVM.

📈

High Demand

Architecture of Java

Java architecture defines how Java programs are created, compiled, and run. It has three main layers: JDK (outermost), JRE (middle), and JVM (innermost).

How Java Code Runs
  • 1 Write source code in a .java file
  • 2 Javac (Java Compiler) compiles it into .class bytecode
  • 3 JVM Class Loader loads the .class file into memory
  • 4 Interpreter converts bytecode line-by-line to machine code
  • 5 JIT Compiler optimises frequently-used code for faster execution
  • 6 Program executes on the hardware. Garbage Collector frees unused memory.
💡 WORA — Write Once, Run Anywhere
Because bytecode runs on the JVM (not directly on hardware), the same .class file runs on Windows, Mac, Linux, or any platform that has a JVM installed.

Topic 04

If Statements

Java provides three forms of the if statement for decision-making. They evaluate Boolean expressions to direct program flow.

1️⃣

Simple if

Executes a block only when the condition is true. No alternative block.

2️⃣

if…else

Provides an alternative (else) block that runs when the condition is false.

3️⃣

if…else if…else

Tests multiple conditions in sequence. The matching block executes.

Program: Compare Two Numbers (Simple if)
Program3.java — Simple if
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        if (a > b) {
            System.out.println("a Is Greater");
        }
        if (b > a) {
            System.out.println("b Is Greater");
        }
        if (a == b) {
            System.out.println("a and b are equal");
        }
    }
}
▶ OUTPUT
b Is Greater

Topic 05

Switch Case Statement

The switch statement tests an expression against multiple case values. When a match is found the corresponding block executes. A break statement ends the switch. Without it, execution falls through to the next case. A default block runs when no case matches.

Switch Syntax
switch (expression) {
    case value1:
        // code to execute
        break;    // optional — stops fall-through
    case value2:
        // code to execute
        break;
    default:
        // code if no case matches
}
Program: Module Code Lookup
SwitchDemo.java — Module Name from Code
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        System.out.println("Enter Module Code");
        String code = input.nextLine();

        switch (code) {
            case "ITU07313":
                System.out.println("Data Structure and Algorithm");
                break;
            case "ITU07312":
                System.out.println("Programming in Java");
                break;
            case "ITU07212":
                System.out.println("Programming in C++");
                break;
            default:
                System.out.println("Unknown Module");
        }
    }
}
▶ OUTPUT (sample — entering ITU07312)
Enter Module Code ITU07312 Programming in Java
⚠️ Fall-Through Warning
If you omit break, Java continues executing subsequent cases even if they don't match. This can cause unexpected results. Always add break unless fall-through is intentional.

Topic 06

For Loop & While Loop

🔢

for loop

Best when the number of iterations is known. init, condition, and increment are all inside the header.

🔄

while loop

Best when iterations depend on a condition. Init is outside, condition is checked before each iteration.

⤵️

do-while loop

Executes the body at least once. Condition is checked after the first execution.

for Loop — Syntax & Examples
ForLoop.java — Print 1 to 5
// Syntax: for (init; condition; incr/decr)

for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}
▶ OUTPUT
1 2 3 4 5
Stars.java — Triangle Pattern (Nested for loop)
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("*");
    }
    System.out.print("\n");
}
▶ OUTPUT
* ** *** **** *****
InvertedTriangle.java — Inverted Star Pattern
for (int i = 0; i <= 8; i++) {
    for (int j = 1; j <= 8 - i; j++) {
        System.out.print("*");
    }
    System.out.print("\n");
}
▶ OUTPUT
******** ******* ****** ***** **** *** ** *
while Loop — Syntax & Example
WhileLoop.java — Print 1 to 5
int n = 1;
while (n <= 5) {
    System.out.println(n);
    n++;    // increment must be inside the loop body
}
▶ OUTPUT
1 2 3 4 5
do-while Loop — Syntax & Example
DoWhile.java — Always executes at least once
int i = 1;
do {
    System.out.println(i);
    i++;
} while (i <= 5);  // note: semicolon after while
Key Difference
for / while: condition checked before the first iteration — body may never run.
do-while: condition checked after the first iteration — body always runs at least once.

Topic 07

Break & Continue Statements

🛑

break

Immediately exits the loop (or switch). No more iterations occur after a break.

⏩

continue

Skips the remainder of the current iteration and jumps to the next iteration.

Program: break — Stop Loop at Iteration 3
BreakDemo.java
for (int i = 1; i <= 8; i++) {
    System.out.println("Iteration No " + i);
    if (i == 3) {
        break;   // exits loop when i equals 3
    }
}
▶ OUTPUT
Iteration No 1 Iteration No 2 Iteration No 3
Program: continue — Skip Iterations 4 and 5
ContinueDemo.java — Print 1,2,3,6,7,8 (skip 4 and 5)
for (int i = 1; i <= 8; i++) {
    if (i == 4 || i == 5) {
        continue;  // skip printing 4 and 5
    }
    System.out.println(i);
}
▶ OUTPUT
1 2 3 6 7 8

Program: Demonstrating Variable Types
VariableTypes.java
package AllPrograms;
public class VariableTypes {

    // ── INSTANCE variable (belongs to object)
    int instanceVar = 100;

    // ── STATIC variable (belongs to class)
    static String schoolName = "CBE";

    void showLocal() {
        // ── LOCAL variable (only lives inside this method)
        int localVar = 50;
        System.out.println("Local: "    + localVar);
        System.out.println("Instance: " + instanceVar);
        System.out.println("Static: "   + schoolName);
    }

    public static void main(String[] args) {
        VariableTypes obj = new VariableTypes();
        obj.showLocal();

        // Static variable accessible without object:
        System.out.println("Static direct: " + schoolName);
    }
}
▶ OUTPUT

✅ Key Rules Summary
• Local: No default value — you MUST assign before use. No access modifiers allowed.
• Instance: Gets memory when object is created. Accessible via object reference.
• Static: Shared across all objects. Accessible directly without creating an object.
• Constants: Declared with final. Value cannot change after assignment.

Topic 09

Classes & Objects

A class is a blueprint for creating objects — it groups variables (state) and methods (behaviour) together. An object is an instance of a class — a real entity created from the blueprint.

🎨

State

Object properties — represented by variables (e.g., car colour, speed).

⚡

Behaviour

Actions an object can perform — represented by methods (e.g., accelerate, brake).

🏷️

Identity

A unique name that distinguishes one object from another.

Static Method Example
StaticMethod.java — Average with static function + parameters
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Static method — called without creating an object
    public static void average(float m, float g, float e) {
        float avg = (m + g + e) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        float math, geog, engl;
        Scanner input = new Scanner(System.in);

        System.out.println("Enter Mathematics Marks");
        math = input.nextFloat();
        System.out.println("Enter Geography Marks");
        geog = input.nextFloat();
        System.out.println("Enter English Marks");
        engl = input.nextFloat();

        average(math, geog, engl);  // called without object
    }
}
▶ OUTPUT (sample: 10, 20, 40)
The Average is 23.333334
Instance Method Example
InstanceMethod.java — Average via object
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Instance method — requires an object to call
    public void average(float m, float g, float e) {
        float avg = (m + g + e) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        float math, geog, engl;

        System.out.println("Enter Mathematics Marks");
        math = input.nextFloat();
        System.out.println("Enter Geography Marks");
        geog = input.nextFloat();
        System.out.println("Enter English Marks");
        engl = input.nextFloat();

        // Create an object first, then call the method
        AllProgramsClass callavg = new AllProgramsClass();
        callavg.average(math, geog, engl);
    }
}
Constructor Example

A constructor is a special method with the same name as the class and no return type. It is automatically called when an object is created with new.

ConstructorDemo.java — Constructor with parameters
package AllPrograms;
import java.util.Scanner;
public class AllProgramsClass {

    // Constructor — same name as class, no return type
    public AllProgramsClass(float a, float b, float c) {
        float avg = (a + b + c) / 3;
        System.out.println("The Average is " + avg);
    }

    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        float math, geog, engl;
        
        System.out.println("Enter Mathematics Marks"); math = input.nextFloat();
        System.out.println("Enter Geography Marks");  geog = input.nextFloat();
        System.out.println("Enter English Marks");    engl = input.nextFloat();

        // Creating the object calls the constructor automatically
        AllProgramsClass obj = new AllProgramsClass(math, geog, engl);
    }
}
▶ OUTPUT (entering 10, 10, 10)
The Average is 10.0
📌 Constructor Rules
1. Constructor name must be identical to the class name.
2. No explicit return type (not even void).
3. Cannot be abstract, static, final, or synchronized.

Topic 10

Arrays

A Java array is an object that stores elements of the same data type in contiguous memory. It has a fixed size set at creation (instantiation). Arrays start at index 0.

📏

1D Array

Linear storage — elements arranged in a single row. Accessed with one index: arr[i].

🔲

2D Array

Matrix storage — rows and columns. Accessed with two indices: arr[row][col].

1D Array — Declaration & Initialization
Array1D.java — Declaration, Initialization, and Access
// Method 1: Separate declaration and instantiation
int data[];
data = new int[7];
data[0] = 100;
data[1] = 234;
// ... etc

// Method 4: Declare + instantiate + initialise in one line
int data[] = {100, 234, 100, 4, 19, 100, 345};
Program: 1D Array Operations
Array1DProgram.java — Store, Access, Update, Display All
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {

        int data[] = new int[]{100, 234, 100, 4, 19, 100, 345};

        // Access 3rd element (index 2)
        System.out.println("3rd Element: " + data[2]);

        // Update element at index 1 (value 234 → 5000)
        data[1] = 5000;

        // Display all elements with for-loop
        System.out.println("All Elements:");
        for (int i = 0; i <= 6; i++) {
            System.out.println(data[i]);
        }
    }
}
▶ OUTPUT
3rd Element: 100 All Elements: 100 5000 100 4 19 100 345
2D Array — Declaration & Program
Array2DProgram.java — 3×5 Matrix Operations
package AllPrograms;
public class AllProgramsClass {
    public static void main(String[] args) {

        int data[][] = new int[][]{{100,234,100,4,19},
                                     {345,45,67,90,1000},
                                     {45,67,89,69,35}};

        // Display 3rd element (index [0][2])
        System.out.println("3rd Element: " + data[0][2]);

        // Display element at [2][0]
        System.out.println("data[2][0]: "  + data[2][0]);

        // Update 1000 → 5000 at [1][4]
        data[1][4] = 5000;

        // Display all with nested for-loop
        System.out.println("All Elements:");
        for (int i = 0; i <= 2; i++) {
            System.out.println("Row " + (i+1));
            for (int j = 0; j <= 4; j++) {
                System.out.print(" " + data[i][j]);
            }
            System.out.println();
        }
    }
}
▶ OUTPUT
3rd Element: 100 data[2][0]: 45 All Elements: Row 1 100 234 100 4 19 Row 2 345 45 67 90 5000 Row 3 45 67 89 69 35
✅ Array Quick Reference
• Arrays start at index 0, not 1
• Size is fixed after creation — use ArrayList for dynamic sizing
• 2D array data[3][5] = 3 rows, 5 columns = 15 total elements
• Access with data[row][col] for 2D arrays

Topic 11

Key Terms & Definitions

All essential vocabulary from the Java SE course, each with a definition and a short code example.

Java Fundamentals
Java Language
A high-level, object-oriented, platform-independent programming language created by James Gosling in the early 1990s. It follows the WORA principle — Write Once, Run Anywhere.
▸ Example
public class Hello { public static void main(String[] args) { System.out.println("Hello, Java!"); } }
JDK Toolkit
The Java Development Kit is a software toolkit used to develop and execute Java programs. It contains the Java compiler (javac), debugger (jdb), archiver (jar), JConsole, and the JRE.
▸ Example
// Compile using JDK's javac tool (command line): javac HelloWorld.java // Run using JDK's java tool: java HelloWorld
JRE Runtime
The Java Runtime Environment provides the environment needed to execute Java applications. It includes the JVM, class libraries (AWT, Swing, JDBC), and deployment tools — but NOT the compiler.
▸ Example
// JRE contains libraries you import, e.g.: import java.util.Scanner; // utility library (part of JRE) import java.sql.*; // JDBC database library (part of JRE)
JVM Virtual Machine
The Java Virtual Machine is an abstract machine that converts Java bytecode into machine-specific instructions. It is responsible for memory management (heap, stack), garbage collection, and enabling platform independence.
▸ Example
// .java ──javac──▶ .class (bytecode) ──JVM──▶ machine code // The same .class file runs on Windows, Linux, Mac via their JVMs
Bytecode Compiled Output
The intermediate, platform-neutral code produced by the Java compiler (javac). Stored in a .class file. The JVM interprets or JIT-compiles bytecode into native machine code at runtime.
▸ Example
// After running: javac Message.java // A file Message.class is created — this is the bytecode file // You cannot read it directly; the JVM executes it.
Garbage Collector Memory
An automatic memory management component inside the JVM. It finds objects in the heap that are no longer referenced by any part of the program and frees that memory — preventing memory leaks without programmer intervention.
▸ Example
String s = new String("CBE"); // object created in heap s = null; // original "CBE" object now unreachable // GC will automatically free the memory used by "CBE"
JIT Compiler Optimisation
The Just-In-Time Compiler monitors which bytecode segments are executed most frequently and compiles those directly to native machine code, so the interpreter does not re-translate them. This improves runtime performance.
▸ Example
// A loop executed 10,000 times: for (int i = 0; i < 10000; i++) { ... } // JIT detects this hot spot and compiles it to native code // so subsequent iterations run at near-native speed.
Data Types & Variables
Data Type Type System
Specifies the kind of value a variable can hold and how much memory it occupies. Java has two categories: Primitive (byte, short, int, long, float, double, boolean, char) and Non-primitive (String, Arrays, Classes).
▸ Example
int age = 21; // primitive double gpa = 4.7; // primitive String name = "Juma"; // non-primitive (object) int[] scores = {80,90,70};// non-primitive (array)
Variable Memory
A named memory location whose stored value can change during program execution. Variables must be declared with a data type before use. The name must start with a letter, underscore, or $ — not a digit.
▸ Example
int studentAge = 20; // declared and initialised studentAge = 21; // value changed — variable // Invalid names: 1name, student age, #score
Constant (Final Variable) Memory
A variable declared with the final keyword. Its value is set once and cannot be changed afterwards. By convention, constant names are written in UPPER_CASE.
▸ Example
final double PI = 3.14159; final int MAX_STUDENTS = 50; // PI = 3.0; ← COMPILE ERROR: cannot assign to final variable
Local Variable Scope
Declared inside a method, constructor, or block. It exists only within that block. No default value — you must initialise it before use. Cannot have access modifiers (public, private, etc.).
▸ Example
void calculate() { int result = 0; // local — only lives inside calculate() result = 5 + 3; System.out.println(result); } // result no longer exists after here
Instance Variable Scope
Declared inside a class but outside any method, without the static keyword. Gets memory when an object is created. Has default values (0 for numbers, false for boolean, null for objects).
▸ Example
class Student { String name; // instance variable — default: null int age; // instance variable — default: 0 } // Each object gets its OWN copy of name and age
Static Variable Scope
Declared with the static keyword inside a class. Shared by all objects of that class. Can be accessed directly without creating an object.
▸ Example
class University { static String name = "CBE"; // shared by ALL objects } // Access without object: System.out.println(University.name); // prints: CBE
Object-Oriented Programming
Class OOP
A blueprint or template for creating objects. A class can contain variables (fields), methods, constructors, and inner classes. Every Java program must have at least one class.
▸ Example
public class Car { String colour; // field int speed; // field void accelerate() { speed += 10; } // method }
Object OOP
An instance of a class — a real entity in memory with its own state (variable values) and behaviour (methods). Created using the new keyword.
▸ Example
Car myCar = new Car(); // create object from Car class myCar.colour = "Red"; // set state myCar.accelerate(); // call behaviour
Constructor OOP
A special method that has the same name as the class and no return type. It is automatically invoked when an object is created with new. Used to initialise the object's state.
▸ Example
class Student { String name; public Student(String n) { // constructor this.name = n; } } Student s = new Student("Ahmed"); // constructor called here
Method OOP
A named block of code inside a class that performs a specific task when called. Methods promote code reuse — write once, call many times. May accept parameters and return a value.
▸ Example
public int add(int a, int b) { // method with return value return a + b; } int result = add(5, 3); // result = 8
Static Method OOP
A method declared with the static keyword. Can be called without creating an object. Can only directly access other static members. The main() method is always static.
▸ Example
public static void greet() { System.out.println("Welcome to CBE!"); } // Called without object: greet(); // or ClassName.greet();
Instance Method OOP

The practice of bundling data (variables) and the methods that operate on that data into a single unit (class), and restricting direct access by declaring variables private. Access is granted through public getter/setter methods.

Section A — Theory / Short Answer
Q1 Theory Briefly explain the history of Java programming language and how it got its name. 5 marks ▶
✅ Model Answer

Java was created in the early 1990s by James Gosling, along with Mike Sheridan and Patrick Naughton at Sun Microsystems. The team initially aimed to build a language for digital devices such as set-top boxes and televisions.

They attempted to use C++ but abandoned it due to high memory usage and platform dependency. In 1991 they started the "Green" project, developing a language called Greentalk (.gt), later renamed OAK — named after an oak tree outside Gosling's office.

Because "OAK" was already trademarked by Oak Technologies, the team renamed it to Java in 1995 — a type of coffee from Java Island, Indonesia that Gosling was drinking at the time. This is why Java's logo is a cup of coffee. Oracle later acquired Sun Microsystems in 2010, gaining control of Java, which now runs on over 3 billion devices.

Q2 Theory With the aid of a diagram, explain the Java Architecture (JDK, JRE, JVM) and how a Java program is compiled and executed. 10 marks ▶
✅ Model Answer

Java Architecture consists of three main components:

1. JDK (Java Development Kit) — the outermost layer; contains the Java compiler (javac), debugger (jdb), archiver (jar), JConsole, and the JRE. It is used by developers to write and compile Java programs.

2. JRE (Java Runtime Environment) — contained within the JDK. Provides the environment to run Java applications. Includes the JVM plus class libraries (JDBC, AWT, Swing, JNDI) and deployment tools. End users need only the JRE to run programs.

3. JVM (Java Virtual Machine) — the innermost layer. Converts bytecode into native machine code. Contains: Class Loader, Method Area, Heap, Stack, Native Stack, Native Interface, and the Execution Engine (Interpreter + JIT Compiler + Garbage Collector).

Execution process: Source (.java) → javac compiles → bytecode (.class) → Class Loader loads → JVM Interpreter/JIT converts → Machine code executes.

Because the JVM abstracts the hardware, the same .class file runs on any platform — this is called WORA (Write Once, Run Anywhere).

Q3 Short Answer Define the following terms with examples: (a) Variable (b) Constant (c) Data Type (d) Array (e) Package 10 marks ▶
✅ Model Answer

(a) Variable — A named memory location whose value can change during program execution.
int age = 20; — age can later be reassigned to 21.

(b) Constant — A named memory location declared with final whose value cannot change after assignment.
final int MAX = 100; — assigning MAX = 200 causes a compile error.

(c) Data Type — Specifies the type of value a variable stores and the memory it uses.
double gpa = 4.7; — double stores decimal numbers using 8 bytes.

(d) Array — A fixed-size collection of elements of the same data type stored contiguously, accessed by index starting at 0.
int[] scores = {80, 90, 70}; — scores[0] is 80.

(e) Package — A namespace grouping related classes to prevent naming conflicts and organise code.
package AllPrograms; — groups all classes in the AllPrograms package.

Q4 Theory Differentiate between the following pairs: (a) JDK vs JRE (b) for loop vs while loop (c) break vs continue (d) static method vs instance method 8 marks ▶
✅ Model Answer

(a) JDK vs JRE
JDK (Java Development Kit) is used to develop Java programs — it includes the compiler, debugger, and the JRE. JRE (Java Runtime Environment) is used to run Java programs only — it contains the JVM and libraries but NOT the compiler. JDK ⊃ JRE.

(b) for loop vs while loop
In a for loop the init, condition, and increment are all enclosed in the loop header — best when iteration count is known. In a while loop they are separate — best when iterations depend on a dynamic condition. Both check the condition BEFORE each iteration.

(c) break vs continue
break exits the loop entirely — no further iterations occur. continue skips the rest of the current iteration and jumps straight to the next iteration; the loop keeps running.

(d) Static method vs Instance method
A static method is declared with static and can be called without creating an object. An instance method has no static keyword and requires an object to be created first before it can be called.

Q5 Theory State FIVE reasons why Java is widely used as a programming language. 5 marks ▶
✅ Model Answer

1. Robust — Java has built-in error-handling features such as try-catch blocks that reduce runtime errors and make programs safer.

2. Platform Independent (WORA) — Java programs compile to bytecode that runs on any platform with a JVM, without rewriting the source code.

3. Object-Oriented — Supports encapsulation, inheritance, and abstraction, promoting code reuse and clear program structure.

4. Automatic Garbage Collection — The JVM automatically frees memory no longer in use, preventing memory leaks without programmer intervention.

5. Free and Open-Source — Java is completely free to use, reducing development costs and encouraging widespread adoption.

Q6 Theory Explain the three types of variables in Java (Local, Instance, Static) with their characteristics. 9 marks ▶
✅ Model Answer

1. Local Variable — Declared inside a method, constructor, or block. Characteristics: (i) accessible only within its declaring block; (ii) no default value — must be initialised before use; (iii) cannot have access modifiers; (iv) the static keyword cannot be used.

2. Instance Variable — Declared inside a class but outside any method, without static. Characteristics: (i) memory allocated when an object is created; (ii) has default values (0, false, null); (iii) each object gets its own separate copy; (iv) access modifiers can be applied.

3. Static Variable — Declared with the static keyword inside a class. Characteristics: (i) memory allocated once when the class loads; (ii) shared by ALL objects of the class; (iii) accessible directly without creating an object; (iv) has default values.

Q12 Practical Implement a Java class named "BankAccount" that demonstrates encapsulation. The class should have a private variable "balance", a method to deposit, a method to withdraw (reject if insufficient), and a method to display the balance. 15 marks ▶

class BankAccount {
    private double balance = 0;  // encapsulated — hidden from outside

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: " + amount);
        } else {
            System.out.println("Invalid deposit amount");
        }
    }

        if (amount > balance) {
            System.out.println("Insufficient funds");
        } else {
            balance -= amount;
            System.out.println("Withdrawn: " + amount);
        }
    }

    public void displayBalance() {
        System.out.println("Current Balance: " + balance);
    }
}

public class AllProgramsClass {

        BankAccount acc = new BankAccount();
        acc.deposit(5000);
        acc.withdraw(2000);
        acc.withdraw(4000); // should fail
        acc.displayBalance();
    }
}
